# 智学伴 · 验收标准（A+B 可运行网站）

> **版本**：v1.0 · 2026-07-20  
> **依据**：`智学伴_功能大纲_AB版.md`、`智学伴_PRD.md`（以 AB 裁剪为准）、`智学伴_原型v3.html`  
> **目标**：交付**真正可运行**的师生 Web 应用，主路径可演示闭环；非原型截图站。  
> **学科钉死**：高中数学（示范学科）；不做全科通用框架。

---

## 0. 总判定规则

| 级别 | 含义 | 发布门槛 |
|------|------|----------|
| **P0** | 阻塞；主路径或缺一不可模块 | **全部必须通过** |
| **P1** | 重要体验/完整性 | 允许 ≤2 项已知问题并有 workaround |
| **P2** | 增强项 | 可延后，不挡演示 |

**通过定义**：功能在本地/部署环境可操作，有可见结果；演示账号 + 预置数据可在 **5 分钟内**走完主路径，无需改代码。

**UI 基线**：蓝主色 `#2563EB`、卡片式布局、师生角色切换；风格对齐 `智学伴_原型v3.html`，不另起皮肤。

## 1. 主路径验收（P0 · 必须端到端）

### 1.1 路径一 · 月考后闭环（主演示 3–5 分钟）

| # | 步骤 | 可勾选标准 | 级别 |
|---|------|------------|------|
| M1 | 演示登录/进入系统 | 存在教师、学生演示身份；可进入对应端（登录或一键角色切换均可，但身份上下文正确） | P0 |
| M2 | T1 加载学情 | 打开学情诊断即见：**热力矩阵、薄弱 Top5、趋势、分层概览**（可用预置/导入成绩） | P0 |
| M3 | Top5 → 分层 | 点击某薄弱知识点（如「函数单调性」）的「分层/出练」，进入分层教学且**知识点已选中** | P0 |
| M4 | 生成三档练习 | 一次生成 **A/B/C 三栏**；每栏至少含：题干、答案要点/参考、难度或目标说明 | P0 |
| M5 | 推送作业 | 教师确认推送后：写入作业任务 + **站内通知**（学生侧可见） | P0 |
| M6 | 学生做作业 | 学生端 S3 分层作业列表出现对应任务；可进入作答并**提交** | P0 |
| M7 | 批改出结果 | 教师 T3 可通过「演示批改 / 结构化导入 / 上传附件+出报告」任一方式得到**完整批改报告** | P0 |
| M8 | 批改通知 | 批改完成后学生收到「批改完成」类通知，可跳转查看结果/评语 | P0 |
| M9 | 档案下钻 | 从 T1 分层名单或 T4 学生表点姓名进入 T5，展示该生档案（档次、薄弱、趋势/时间线等至少 3 类信息） | P0 |

**路径一总判**：M1–M9 连续操作无死链、无空白关键页；数据在师生两侧可回显。

### 1.2 路径二 · 学生自主补弱

| # | 步骤 | 可勾选标准 | 级别 |
|---|------|------------|------|
| S-P1 | S1 看薄弱 | 我的学情展示个人档次 + 薄弱知识点/建议 | P0 |
| S-P2 | S2 每日练 | 展示 1–3 题；客观题（选择/填空）提交后**即时对错** | P0 |
| S-P3 | 错题入库 | 做错的题进入 S4 错题本 | P0 |
| S-P4 | 成长轨迹 | 完成每日练或作业后，S5 出现对应事件/打卡（预置+新事件均可） | P1 |

### 1.3 路径三 · 开班配置

| # | 步骤 | 可勾选标准 | 级别 |
|---|------|------------|------|
| C1 | 新建班级 | T4 可新建班级（名称/年级/学科等最少字段） | P0 |
| C2 | 导入/添加学生 | 支持手动添加 **或** 表格导入至少一种；名单表可见 | P0 |
| C3 | 回诊断选班 | 学情诊断可按班级切换/刷新展示（演示班数据可用） | P0 |

## 2. 模块验收清单

### 2.1 全局壳与导航（P0）

- [ ] **教师顶栏**：`学情诊断 | 分层教学 | 智能批改 | 班级管理 | 🔔 | 教师⇄学生 | 身份名`
- [ ] **学生顶栏**：`我的学情 | 每日练 | 分层作业 | 错题本 | 成长轨迹 | 🔔 | 教师⇄学生 | 身份名`
- [ ] **无**全局 AI 悬浮球、**无**「AI 学伴」导航、**无**「知识管理」主 Tab（或已从主导航移除）
- [ ] 师生切换后导航与首页内容同步切换，不串数据身份
- [ ] 主色/卡片/分层 A 紫 B 蓝 C 绿视觉与 v3 一致（允许实现细节差异，不可换肤成其他品牌色）

### 2.2 学情诊断 T1（P0/P1）

| 项 | 标准 | 级别 |
|----|------|------|
| 热力 | 章节/知识点得分率色块（五档色或等价映射）可渲染 | P0 |
| Top5 | 班级得分率最低 5 个知识点 + 快捷「分层/出练」 | P0 |
| 趋势 | 近若干次考试/测验折线（演示数据可） | P0 |
| 分层概览 | A/B/C 人数；可展开名单 | P0 |
| 规则分层 | 分位规则（如 20%/60%/20%）或等价规则；**教师可手调**学生档次 | P0 |
| 数据输入 | Excel/表格导入成绩 **或** 一键加载演示成绩 | P0 |
| 题点映射 | 题号—知识点—满分有预置模板；可查看/基础编辑即可 | P1 |
| 禁止表述 | 界面**不承诺** K-Means 聚类算法、Neo4j 真图谱引擎 | P0 |

### 2.3 分层教学 T2（P0/P1）

| 项 | 标准 | 级别 |
|----|------|------|
| 生成 | 选知识点 + 课型 → 生成 A/B/C **练习**（非完整课件工厂） | P0 |
| 内容结构 | 每档：目标说明 + 若干题（题干+答案要点+难度说明） | P0 |
| 编辑 | 单题可编辑（改题干/答案至少一项） | P0 |
| 页内微调 | 支持「换一题」**或**短指令微调（如加题/调难度）；**仅工坊内**，非全局聊天 | P1（有 AI 时）/ P0 至少有本地编辑 |
| AI 出题 | 配置读取本机 Codex（`~/.codex/config.toml` + `auth.json`）；`base_url=https://passion8.cc/v1`，`model=gpt-5.5` | P0（集成） |
| 失败兜底 | 模型失败时用题模版/预置题填充，页面不白屏 | P0 |
| 导出示意 | 复制或下载至少一种可用 | P1 |
| 推送 | 按档推送给对应分层学生 → 作业列表 + 通知 | P0 |
| 禁止 | **无**完整 8–15 页自动课件工厂；**无**全局备课对话 | P0 |

### 2.4 智能批改 T3（P0 · 演示/结构化）

| 项 | 标准 | 级别 |
|----|------|------|
| 入口 | 上传文件 **或** 导入对错/成绩表 **或**「加载演示批改结果」至少两种 | P0 |
| 总览 | 提交率、均分、正确率（或等价班级汇总） | P0 |
| 错因 | 五类分布：概念 / 计算 / 审题 / 方法 / 表达 | P0 |
| 个体 | 逐题对错摘要 + 可读评语 | P0 |
| 评语 AI | 可单次生成个性化评语（模板+模型）；失败有规则评语兜底 | P1 |
| 错题同步 | 错误项写入学生错题本 | P0 |
| 通知 | 批完通知学生 | P0 |
| 禁止 | **不承诺**生产级手写 OCR 准确率；图片可仅作附件展示 | P0 |

### 2.5 班级管理 T4（P0）

- [ ] 班级列表卡片（人数、学科/学期等基础信息）
- [ ] 新建班级
- [ ] 学生名单表；添加/导入学生
- [ ] 点姓名进档案
- [ ] 分层标签在名单中可见（与规则分层一致）
- [ ] 课代表仅「标记」即可，**无**独立权限后台（P2 可选标记）

### 2.6 学生档案 T5（P0/P1）

- [ ] 头信息：姓名、班级、当前档、学号（有则显示）
- [ ] 简易能力图/个人热力、错因、近次成绩、时间线中 **≥3 块**有真实或演示数据
- [ ] 操作：调整分层、推送练习
- [ ] **单次**生成诊断段落（按钮生成一段结论，**非**多轮聊天页）
- [ ] 入口：分层名单 / 班级名单 /（可选）批改列表、预警通知

### 2.7 通知中心（P0）

| 角色 | 类型（至少实现带 * 的） | 级别 |
|------|-------------------------|------|
| 学生 | *新作业、*批改完成、每日练提醒、分层变动 | P0 前两项；后两项 P1 |
| 教师 | *作业提交、全员提交、学情预警（规则） | P0 至少提交类；其余 P1 |

- [ ] 顶栏铃铛 + 未读角标
- [ ] 列表可「已读」；点击跳转业务页
- [ ] **仅站内**；无微信/短信

### 2.8 学生端模块

| 模块 | 验收要点 | 级别 |
|------|----------|------|
| S1 我的学情 | 个人热力/掌握、档次、薄弱与建议；可详情弹层 | P0 |
| S2 每日练 | 按薄弱 1–3 题；客观题即批；错入错题本；正确记成长事件 | P0 |
| S3 分层作业 | 待完成/已完成；开始做题、看结果 | P0 |
| S4 错题本 | 列表、错因、筛选；变式/重做（规则或预置） | P0 列表；P1 变式 |
| S5 成长轨迹 | 曲线 + 时间轴 + 简单徽章；**不**重度游戏化 | P0 展示；P1 动态徽章 |
| — | **无** AI 学伴页 | P0 |

**定位边界（防重复）**

| 模块 | 视角 | 不得做成 |
|------|------|----------|
| 我的学情 | 当前掌握数据分析 | 不得仅是作业列表 |
| 每日练 | 面向未来推荐题 | 不得与错题本同一列表无区分 |
| 错题本 | 回顾已错 | 不得自动变成每日推题主入口 |
| 成长轨迹 | 时间线激励 | 不得再堆一整页诊断热力替代学情页 |

## 3. AI 使用边界验收（P0）

| 允许 | 禁止 |
|------|------|
| 三档练习出题（JSON 结构化） | 全局 AI 聊天助手（悬浮/全页） |
| 分层工坊内短指令微调 | 学生 AI 学伴连续对话 |
| 批改/档案 **单次**评语或诊断段落 | 开放题无人工的「生产级全自动精批」承诺 |
| 配置：本机 `~/.codex/config.toml` + `auth.json`；`base_url=https://passion8.cc/v1`；`model=gpt-5.5` | 写死其他未约定云厂商密钥进仓库 |

- [ ] 代码与文档中 AI 调用范围与上表一致
- [ ] 密钥不入库；从本机 Codex 配置读取
- [ ] 无网/鉴权失败时有兜底内容，主路径仍可演示

## 5. 数据与实体最小集（实现验收）

系统至少能持久化或稳定演示下列实体关系：

```
Teacher, Class, Student
Exam, ExamScore, QuestionMeta
LayerAssignment
ResourceSheet (A/B/C 练习 JSON)
HomeworkPush, HomeworkSubmission
GradingReport, ErrorRecord
DailyPracticeItem, GrowthEvent
Notification
```

- [ ] 演示班预置数据可一键恢复
- [ ] 推送/提交/批改/通知状态可回读，刷新不丢（本地 DB 或服务端均可）
