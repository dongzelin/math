## 4. 失败项和缺陷

### P0-SEC-001：可直接读取其他学生档案，已修复

| 项目 | 内容 |
|---|---|
| 关联用例 | `XAUTH-001`、`XAUTH-003`、`XAUTH-004`、`XAUTH-006`、`XAUTH-007` |
| 复现步骤 | 直接请求 `GET /api/students/stu-s03/profile`，不提供服务端鉴权凭证。 |
| 期望 | 未登录或非本人身份返回 401/403，不泄露姓名、班级、个人热力、错题和成长信息。 |
| 初测实际 | 返回 200 和完整学生档案。 |
| 影响 | 学生档案、错题、作业、通知、报告等资源目前依赖前端路由限制，直接 API 调用可绕过，存在越权读取和潜在越权写入风险。 |
| 根因定位 | 路由 `server/src/routes/api.js` 未实现会话验证和 `role/userId` 资源归属中间件。 |
| 修复建议 | 登录成功后签发 HttpOnly session 或短期 token；为每个资源路由注入认证上下文；学生仅允许访问 `studentId===session.studentId`，教师仅允许访问其拥有的 `classId`；为所有写接口补 401/403 用例。 |
| 修复实现 | 登录和注册返回 12 小时 Bearer token；token 仅以 SHA-256 哈希保存于 `auth_sessions`；`/api` 默认要求认证。教师按 `teacher_id` 校验班级，学生仅能访问自身 ID，通知、作业、测验、报告和每日练均校验资源归属。 |
| 复测结果 | Pass。真实服务未携带 token 访问学生档案返回 401；学生 token 访问 `stu-s03` 档案返回 403；隔离回归 `XAUTH-001`、`XAUTH-002` 均通过。 |

### P0-AI-001：分层练习 AI 调用超时无降级响应，已修复

| 项目 | 内容 |
|---|---|
| 关联用例 | `API-AI-001`、`BND-I-004`、`UI-L-005` |
| 复现步骤 | POST `/api/ai/layered-sheet`，传入 `kpName=函数单调性`，客户端等待上限 3.5 秒。 |
| 期望 | 在服务端设定超时内返回 A/B/C 模板练习或可读错误，不阻塞页面。 |
| 初测实际 | 3.5 秒客户端超时；未返回 `sheet`。 |
| 影响 | 分层教学的 AI 主入口会长期 loading，模板兜底无法触发。 |
| 根因定位 | `server/src/ai/client.js` 中 `fetch(url, ...)` 未配置 `AbortController` 或超时；异常 catch 只能在上游请求结束后执行。 |
| 修复建议 | 在 AI client 增加 8-15 秒服务端超时，抛出可识别错误；`generateLayeredSheet` 捕获该错误并返回本地模板；前端显示“已切换题库模板”状态。 |
| 修复实现 | `chatCompletion` 使用 `AbortController`，服务端上限 8 秒；超时抛出 `AI_TIMEOUT`，现有分层练习 catch 分支返回模板。 |
| 复测结果 | Pass。隔离回归 `API-AI-001` 在 10 秒上限内返回带 `sheet` 的降级结果。 |

### P0-AI-002：在线测验 AI 出卷超时无降级响应，已修复

| 项目 | 内容 |
|---|---|
| 关联用例 | `API-AI-002`、`BND-I-004`、`UI-QT-007` |
| 复现步骤 | POST `/api/ai/quiz-paper`，题型组成 `{choice:1,fill:1,judge:1,essay:1}`；真实 `8788` 服务客户端超时设为 20 秒。 |
| 期望 | AI 不可用时返回完整四题型本地题库试卷，`fallback=true` 或 `source=template`。 |
| 初测实际 | `20.06` 秒后客户端请求超时，未返回试卷。 |
| 影响 | 产品核心“AI 出卷 -> 发布 -> 学生作答”路径在当前上游不可用时被阻塞。 |
| 根因定位 | 同 P0-AI-001，且 `generateQuizPaper` 的模板 catch 依赖 AI fetch 先结束或抛错。 |
| 修复建议 | 复用 AI client 超时；增加出卷 endpoint 的超时回退测试；将超时视为可恢复错误并立即走 `fallbackByType`。 |
| 修复实现 | 复用 8 秒 AI 超时；在线测验首次出现网络或超时错误后，剩余题型直接调用本地 `fallbackByType`，不再逐题型继续等待 AI；所有非纯 AI 结果均返回 `fallback=true`。 |
| 复测结果 | Pass。真实 `8788` 服务在 `8.035` 秒返回 HTTP 200、4 题、`source=mixed`、`fallback=true`；隔离回归 `API-AI-002` 通过。 |
